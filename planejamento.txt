# Push_swap — Documentação 01

# Visão Geral do Projeto

## Objetivo

Desenvolver um sistema capaz de ordenar uma sequência de números inteiros utilizando apenas as operações permitidas pelo Subject.

O sistema deverá receber os dados pela linha de comando, processá-los utilizando a estratégia de ordenação apropriada e produzir como saída uma sequência de operações capaz de ordenar a pilha A.

---

# Escopo

O projeto é composto por um conjunto de módulos interdependentes que trabalham em sequência para transformar uma entrada em uma solução válida.

Fluxo geral do sistema:

```
Entrada
    ↓
Validação
    ↓
Estruturas de Dados
    ↓
Análise (Disorder)
    ↓
Seleção da Estratégia
    ↓
Estratégia de Ordenação
    ↓
Operações
    ↓
Saída
```

Módulos auxiliares:

* Tratamento de Erros
* Benchmark
* Avaliação de Desempenho

---

# Arquitetura Geral

O sistema é dividido em onze módulos principais.

1. Entrada de Dados
2. Validação
3. Estruturas de Dados
4. Operações
5. Disorder
6. Seleção da Estratégia
7. Estratégias de Ordenação
8. Saída
9. Benchmark
10. Tratamento de Erros
11. Desempenho

Cada módulo possui uma responsabilidade específica dentro do sistema e depende apenas dos módulos necessários ao seu funcionamento.
